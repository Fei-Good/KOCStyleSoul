一、产品背景与目标
1.1 背景
微信视频号、抖音、小红书等平台上存在大量普通 KOC，他们有真实表达欲望，却面临三大痛点：
- 定位模糊：不清楚自己的内容风格是什么、优势在哪
- 选题低效：盲目追热点，内容同质化，无法形成差异化
- 运营不专业：不知道发什么时间、怎么互动、怎么持续成长
现有工具（如热榜推送、模板工具）的逻辑是"先喂热点再让你跟风"，导致 KOC 失去个人特色，涨粉效率低且难以持续。
1.2 核心洞察
每个KOC 个体都有独特的风格基因，爆款不是模仿出来的，而是从自身 DNA 中进化出来的。

    ↓
【绑定 B 站主页 URL】（必填，无兜底，作品发布自动更新）

【进入工作台首页】

    ↓
【选一个选题 + 选几张技能卡 → 去创作】
    ↓
【发布 → 数据回流 → DNA 自动更新】
    ↓
（每天打开首页，看热点+选题，循环成长）

七、数据流与技术架构概要
数据流：

B站主页/视频 URL
    └── B 站公开接口抓取
          └── 多模态内容分析（标题+封面+弹幕+互动数据）
                ├──→ 风格 DNA 向量（用户画像库）
                └──→ 技能卡结构化输出（个人技能知识库）

每日热点数据
    └── B 站热门榜抓取（每日 08:00）
          └── 与用户 DNA 向量匹配
                └──→ 首页热点适配度 + 专属选题生成

技能卡导出接口
    └── JSON / Markdown → 剪辑 Agent / 脚本 Agent（下游）
系统模块：

前端层    H5（微信内嵌）+ PC Web
          核心页面：首页 / DNA页 / 技能卡库 / 诊断

AI层      风格 DNA 解码  ── 多模态视频理解
          技能卡提取    ── 结构化知识抽取
          选题生成      ── DNA × 热点 → 个性化推荐
          内容诊断      ── 结构化评分

数据层    用户 DNA 向量库（向量存储，定期更新）
          个人技能卡知识库（结构化存储，可导出）
          热点数据库（每日更新）

外部依赖  B 站公开接口（视频/主页数据）
          热榜数据源（B 站全站日榜）
